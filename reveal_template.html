Analisis Segementasi dan Prediksi Churn Pengguna Spotify Berbasis Data Spotify Customer Churn Menggunakan Algoritma Random Forest

Proyek Data Mining - Kelompok 12

Rival Fitrah Dermawa, Hotniel Diasi Sianturi, Anwar Faiz Fauzi

2025-11-19

Latar Belakang Masalah

  • Pelanggan dapat dengan mudah menggunakan haknya untuk berpindah penyedia layanan dari satu aplikasi ke aplikasi lainnya.
  • Churn pelanggan mengacu pada kehilangan pelanggan berkala dalam sebuah organisasi.
  • Aktivitas churn sangat berpengaruh terhadap total profit dan citra bisnis, sehingga sebaiknya dapat diprediksi dan dicegah.
  • Data mining mengidentifikasi churnes lebih awal sebelum mereka pindah.

Sumber dan Karakteristik Data

  • Sumber: Dataset publik Spotify Customer Churn (Kaggle)
  • Dasar: Berisi informasi demografi, perilaku mendengarkan, dan status langganan pengguna.

Karakteristik Data

  • Jumlah Data: 1,000 kombinasi unik (kategori)
  • Struktur: File CSV
  • Tipe Data: Campuran kategorikal & numerik

Variabel Dalam Dataset (1/2)

Informasi Langganan:

  • subscription_type (kategorikal)
  • country (kategorikal)

Aktivitas Mendengarkan:

  • avg_daily_minutes (numeric)
  • number_of_playlists (integer)
  • top_genre (kategorikal)
  • skips_per_day (integer)

Variabel Dalam Dataset (2/2)

Aktivitas di Aplikasi:

  • support_tickets (integer)
  • days_since_last_login (integer)

Target:

  • churned (Churn/ Aktif)

Kualitas Data

  • Tidak terdapat missing value.
  • Tidak ada data korup atau format tidak konsisten.
  • Cocok untuk analisis prediktif.

Metodologi Penelitian

Metodologi Penelitian

Penelitian ini terdiri dari 5 tahapan:

  1. Business Understanding: Memahami permasalahan bisnis yang ingin diselesaikan.
  2. Data Understanding: Analisis awal berupa statistik deskriptif
  3. Data Preparation: Pembersihan dan transformasi fitur.
  4. Modeling: Pemodelan klasifikasi menggunakan algoritma Random Forest.
  5. Evaluasi: Mengevaluasi model menggunakan metrik performa

Tahap 1 Business Understanding

  • Memahami bagaimana pelanggan Spotify dapat berhenti berlangganan berdasarkan perilaku dan pola penggunaan aplikasi

  • Churn dapat mempengaruhi pendapatan dan stabilitas pelanggan Spotify

Tahap 2 Data Understanding

  • Analisis struktur data (str, summary).

  • Deteksi missing value & anomali.

  • Eksplorasi pola perilaku pengguna.

  • Visualisasi distribusi dan hubungan variabel.

Tahap 3 Data Preparation

  • Menghapus/isi missing value.

  • Menghilangkan duplikasi.

  • Encoding variabel kategorikal → numerik.

  • Normalisasi/standarisasi fitur numerik.

  • Seleksi fitur yang relevan.

Tahap 4 Modeling

  • Algoritma yang dipakai: Random Forest.
  • Alasan Pemilihan:

    • Stabil

    • Tahan terhadap overfitting

    • Mampu menangani banyak fitur

  • Data Bagi: (70% training dan 30% test)

Tahap 5 Evaluasi Model

Metrik yang digunakan:

  • Akurasi
  • Precision
  • Recall
  • F1-score
  • Confusion Matrix

Teknik dan Algoritma

Random Forest

Mengolah berbagai atribut perilaku pengguna Spotify dengan hasil yang akurat dan stabil.

Rencana Implementasi

Pengumpulan dan Pemahaman Data

  • Mengunduh dataset
  • Memuat dataset ke dalam RStudio
  • Meninjau struktur data
  • Menganalisis karakteristik awal data
  • Menentukan fitur yang berpotensi memengaruhi status churn pengguna

Pembersihan dan Pra-pemrosesan Data

  • Menangani missing values
  • Menghapus duplikasi data
  • Mengubah tipe data
  • Melakukan normalisasi atau standarisasi
  • Menghapus kolom yang tidak relevan

Eksplorasi dan Analisis Data

  • Membuat visualisasi distribusi data menggunakan ggplot2.
  • Menganalisis hubungan antara perilaku pengguna
  • Mengidentifikasi adanya outlier
  • Melakukan analisis korelasi antar fitur

Seleksi Fitur (Feature Selection)

  • Menentukan fitur signifikan.
  • Menghapus fitur korelasi rendah atau berlebihan (redundan).
  • Memastikan semua fitur sesuai format input untuk algoritma Random Forest.

Pembangunan model prediksi

  • Membagi dataset menjadi data latih (70%) dan data uji (30%)
  • Melatih model Random Forest.
  • Melakukan tuning parameter
  • Menyimpan model terbaik untuk tahap evaluasi.

Evaluasi

Evaluasi dilakukan untuk menilai akurasi dan performa model dalam memprediksi churn

Hasil

  1. Adanya segmentasi dari pengguna yang akan merepresentasikan pola dari perilaku pengguna.
  2. Mengidentifikasi faktor-faktor penyebab pengguna churn dengan menggunakan model klasifikasi
  3. Model prediksi churn yang dihasilkan performa baik, khususnya dalam mendeteksi pengguna yang berisiko churn
  4. Visualisasi diharapkan dapat membantu dalam memahami pola dan membuat narasi hasil analisis.

Dampak

Dampak Akademik:

  • Membantu mahasiswa memahami penerapan nyata teknik clustering dan klasifikasi pada data pengguna
  • Memperkuat kemampuan dalam eksplorasi data, preprocessing, modeling, dan evaluasi

Dampak Teknis:

  • Memberikan gambaran tentang bagaimana perilaku mendengarkan musik dapat dikelompokkan dan digunakan untuk memprediksi kemungkinan churn lebih awal.
  • Menyediakan insight sederhana mengenai kebiasaan pengguna yang berisiko churn, segmen yang perlu diperhatikan, serta rekomendasi dasar untuk meningkatkan retensi pengguna.
